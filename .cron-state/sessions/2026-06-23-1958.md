## Shipped

1. **`201382c` — Shared diff-output-bytes constants module**
   `apps/cli/src/diff-output-limits.ts` exports the canonical
   `DIFF_DEFAULT_MAX_OUTPUT_BYTES` (100 KiB) and
   `DIFF_MAX_OUTPUT_BYTES_CEILING` (16 MiB). The four pre-existing
   command-scoped constants on `presets.ts` and `review.ts` are now
   `=` aliases of the canonical exports. A future bump lands in ONE
   place. 9 new tests (diff-output-limits.test.ts) pin the canonical
   values + per-command alias delegation + cross-command agreement.

2. **`9755b36` — `review filter-report --diff --on-delta-once-per <minutes>`**
   TTL variant of tick-27's `--on-delta-once`: fire the `--on-delta`
   hook at most once per N minutes per (base, target) pair. Mutex
   with `--on-delta-once`. New `ON_DELTA_ONCE_PER_FIRED_AT` Map,
   `parseOnDeltaOncePerFlag` pure parser, injectable `now()` clock
   seam. 1-year sanity ceiling rejects ridiculous values with a hint.
   11 new tests covering parser arms + 6 runtime arms (first fire
   records / within-TTL suppressed / after-TTL refire / mutex /
   invalid pre-network reject / per-pair isolation / back-compat).

3. **`0341214` — `review filter-report --diff --input-strict`**
   Opt-in deep envelope validator. The tick-27 shallow parser only
   checks `base`/`target` are objects; the strict mode walks every
   required field on FilterReportBodyFull / FilterReportBodySlim
   with per-field-path error labels. New `validateFilterReportDiffInputBodyStrict`
   + `parseInputStrictFlag` pure helpers + `FilterReportDiffInputStrictResult`
   discriminated union. Validates `base` before `target` for stable
   error labels. Catches a worker writing the wrong wire shape
   BEFORE the diff command produces a misleading delta. 14 new tests
   (boolean sugar + 8 pure validator arms + 5 runtime integration).

4. **`429b78b` — Aggregator `summariseFilterReportDelta` + axis tuple**
   Compact summary helper. Where `FilterReportDelta` carries the
   full per-axis before/after shape, the summary collapses it to:
   - `hasDelta` (mirror),
   - `changedAxes: readonly FilterReportDeltaAxis[]` (frozen tuple
     of just the axis names that flipped, canonical order),
   - `regression` (target is "noisier": input grew / filter removed /
     drop shrank with flat input),
   - `bugFix` (target is "quieter": input shrank / filter added /
     drop grew with flat input).
   Both bits can fire on a mixed change (deliberate: webhook router
   resolves precedence itself). New `FILTER_REPORT_DELTA_AXES` 5-tuple
   pins the canonical order. 12 new tests covering all the bit
   combinations + threshold-only "neither bit fires" case + the
   frozen-array guarantee.

5. **`f8d0c4d` — `presets diff --on-delta` + `--on-delta-template`**
   Symmetric port of tick-26's review filter-report `--on-delta` hook
   to the presets command. Same shell-exec contract, same mutex
   semantics, same JSON payload shape. New `PresetsDiffOnDeltaExecer`
   interface + default impl + `PRESETS_DIFF_ON_DELTA_TEMPLATES` closed
   2-tuple + `expandPresetsDiffOnDeltaTemplate` + `parsePresetsDiffOnDeltaFlags`
   + `runPresetsDiff` gains injectable `{ onDeltaExecer?, env? }`.
   Env-var ladder uses PRESETS_ namespacing on the primaries
   (SLACK_PRESETS_DELTA_WEBHOOK_URL -> SLACK_WEBHOOK_URL) so an
   operator can route review filter-report deltas and preset diffs
   to DIFFERENT channels but share a fallback. Hook fires AFTER
   the exit code is set so a CI consumer reading both sees the same
   state. Failures surface on stderr but don't promote exit 3 -> 2.
   13 new tests covering parser + 3 template-expand arms + 6 runtime
   arms (fire/no-fire/non-zero/throw/mutex/template/no-env/empty/back-compat).
