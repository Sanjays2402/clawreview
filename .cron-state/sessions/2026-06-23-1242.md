## Design notes / decisions

- **fireExit() closure pattern** (Feature 1): refactored the existing
  per-arm `if (metrics) observeReviewFilterReportDiff(...)` into a
  single closure that fires BOTH counter + histogram with the same
  (fetchOk, delta) tuple. Guarantees a downstream consumer never sees
  count-without-duration on any exit arm. Backwards-compatible: the
  histogram is opt-in (only fires when metrics bundle is injected).

- **Star sugar early-exit** (Feature 2): the `*` / `all` / `none`
  aliases are checked BEFORE the comma-split so a typo'd `?fields=*`
  never enters the unknown-field error path. Mirrors /digest's tick-19
  ?slim contract exactly so an operator who learned one route's
  projection rules knows the other.

- **STDOUT_SENTINEL Symbol per command** (Feature 3): used a module-
  local sentinel Symbol (distinct registry key from presets diff's)
  instead of reusing the existing one. Trade-off: more code, but
  decouples a future divergence -- one command gaining "+" for
  stdout-and-also-write wouldn't have to coordinate semantics with
  the other.

- **--json-stream stable line ordering** (Feature 4): the 7-line
  shape (header / 5 axes in declaration order / footer) is pinned by
  an export `FilterReportDiffStreamLine` discriminated union AND a
  pure-helper ordering test. A downstream consumer doing
  `head -1 | jq -r .kind` always reads 'header'; `tail -1 | jq -r .hasDelta`
  always reads the gate bit -- those should never break silently.

- **--on-delta does NOT change exit code on hook failure** (Feature 5):
  the operator wired the hook for notification; a misconfigured webhook
  shouldn't promote the diff command's exit from 3 (delta detected) to
  2 (config error). The diff itself succeeded; only the side-channel
  notification failed. Stderr surfaces the failure for log-grep.
