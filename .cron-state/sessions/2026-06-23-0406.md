## Notes

- 6 commits this tick because Feature 1 split telemetry + server-route wiring into two commits (cleaner blame surface; each is a revertible unit on its own).
- `stats --jsonl --no-footer` from the carried backlog deliberately deferred — the stats CLI surface is already heavy this tick and the feature is a one-flag opt-out that benefits from landing solo so its tests don't get tangled with the existing --jsonl coverage. Carried to tick 25.
- The new histogram observation is tucked inside the route's success arm — explicitly chose NOT to use an onResponse hook because the route's body shape (and therefore the per-shape label) is only resolved INSIDE the handler. The route-handler observation captures the full request->response path uniformly.
- `?fields` and `?slim` are mutex by design: ?slim collapses verbose appliedFilters into a single boolean; ?fields picks an arbitrary subset of the FULL shape. Combining them would force the route to negotiate "does slim win or does fields win?" with no clean answer. Refusing loudly forces the operator to pick one mode.
- `--require-filter` watch-mode semantics: the LAST sample's applied bit determines the exit code. This matches single-shot's "last poll wins" contract and means a CI gate observing a slow rollout converges to the resolved state (early samples with applied=false do NOT trip the gate if the final sample shows applied=true).
- The on-regression-template parser was extracted into `parseOnRegressionFlags` so the compare command's --on-regression handling now flows through ONE source of truth (raw command + template both resolved by the same parser). The old inline parsing is gone; tests pin the new shape.
