3. **CLI: project-local presets under `.clawreview/presets/*.yml`** ‚Äî `bc70294`
   - `loadLocalPresets(cwd)` discovers `*.yml` / `*.yaml` files, basename
     becomes the preset name. Local presets SHADOW built-ins on collision.
   - `mergeWithExtends(raw, { localPresets })` plumbs them into the
     resolver. Inner `extends:` in a local preset is stripped with a
     stderr warning.
   - `clawreview validate` surfaces local-preset names in its error path
     and prints them in its success summary.
   - 8 new tests; full cli config-extends suite 15/15.

4. **CLI: `clawreview lint-config`** ‚Äî `c377e80`
   - Recursive walker (no fast-glob added ‚Äî uses node:fs/promises).
     Skips node_modules, .git, dist, build, .next, .turbo, coverage,
     and the `.clawreview/presets/` subtree.
   - Each match's `extends:` resolves against the SAME (built-in + local)
     namespace `loadConfig` uses at runtime, SCOPED PER FILE'S PARENT
     DIR ‚Äî monorepo-friendly so `apps/web/.clawreview/presets/*.yml`
     resolve only for `apps/web`'s config.
   - `--format text` (per-file pass/fail + summary) or `--format json`
     (machine-readable). Exit 0 / 2 / 3 for ok / invalid / no-match.
   - 11 new tests; full cli suite 73/73.

5. **Server: `/api/internal/webhook/recent` filters** ‚Äî `262279f`
   - `?event=<name>`, `?sinceMs=<unix-ms>`, `?since=<ISO-8601>` (alt),
     `?repo=<full_name>`.
   - Filters AND together, applied BEFORE the limit cap so pagination
     stays correct.
   - `WebhookStore.list` overloaded ‚Äî back-compat with numeric limit.
   - Response now includes `appliedFilters` for caller confirmation.
   - Unparseable `receivedAt` is KEPT (bias toward over-include in
     degraded state).
   - 5 new tests; full webhook-replay suite 11/11.
