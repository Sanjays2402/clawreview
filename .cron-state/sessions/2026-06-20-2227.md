     no-mutation, behaviour parity with `aggregate()` on distinct files).

     Default stays severity; `--by agent|category` leads the output
     with the matched block.

     by-category blocks so users who skip `--by` still see them.
   - `--format json` emits totals / byAgent / byCategory / topFiles /
     totalCostUsd / groupBy. CI workflows can capture an artifact AND
     gate on `--fail-on` in the same step.
   - Renamed the per-agent-execution block to "By agent execution:"
     so it doesn't clash with the new per-agent-findings block.
   - 9 new tests (5 --by + 4 --format json).

3. **`9973372` — cli: clawreview presets show <name>**
   - Second sub-command on top of tick 7's `presets list`.
   - Yaml (default) / json / text formats. Yaml output is
     copy-pasteable into `.clawreview.yml` and carries a `# clawreview
     preset: <name>` header naming the source + extends chain.
   - JSON output includes `{ name, source, extends, shadowsBuiltin,
     fields, body }` so tooling can consume the resolved body alongside
     the metadata.
   - Resolution mirrors `loadConfig`: locals shadow built-ins on name
     collision (annotated in output), `extends:` is recursively
     resolved, cycles / unknown extends surface clear errors.
   - Exit codes: 0 success, 1 unknown preset (lists available), 2
     invalid --format or missing <name>.
   - 8 new tests.

4. **`f1404d6` — server: operator-poll rate-limit class**
   - New limiter class scoped to /api/internal/webhook/recent and
     /api/internal/webhook/stats so chatty dashboards don't eat the
     operator's default per-token budget for rerun / replay work.
   - Registered BEFORE the default per-token limiter so polling hits
     the dedicated bucket first; the default limiter still observes
     the request so a wildly misbehaving dashboard still trips the
     global ceiling.
   - Default budget: 3000/min (5x default per-token).
   - 429 response carries `class: "operator-poll"` + dedicated
     `x-ratelimit-operator-{limit,remaining}` headers.
   - Same `DISABLE_PER_TOKEN_RATE_LIMIT=1` env knob disables both
     classes for integration tests.
   - 7 new tests (2 pure + 5 wired).

5. **`02d0eed` — server: webhook stats byRepo slice**
   - Added `byRepo` totals to /api/internal/webhook/stats with a
     `?topRepos=` cap (default 50, hard ceiling 200) and a `(other)`
     tail collapse so the visible map still sums to `total`.
   - Entries with no `repoFullName` land under `(none)` rather than
     being dropped (installation / marketplace events stay visible).
   - Sort: descending count, alphabetical tiebreak.
   - Honors the existing `?repo=` filter so the slice composes with
     existing filtering.
   - `appliedFilters.topRepos` surfaces the honored value so operators
     can debug clamping.
   - 5 new tests.
