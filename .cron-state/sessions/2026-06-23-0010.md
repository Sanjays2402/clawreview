# Tick 23 session — 2026-06-23 00:10–00:30 PT

## Summary
Shipped 5 features building on tick 22's PersistedFilterReport infrastructure. All
five compose around the same theme: making the persisted filter report (worker's
write-time snapshot of which findings the pre-bucket filter dropped and why)
first-class on every consumer surface — server endpoint, CLI face, dashboard
observability, and CI-gateable streaming.

## Features (5/5 — all genuinely solid)

1. **Server `/api/reviews/:id/filter-report` standalone endpoint** (`fa2c58e`) — 
   Lightweight pair-of-three with /api/reviews/:id and /digest. Dashboard "filtered
   N of M" badge can now read the smallest possible payload. `?slim=true` collapses
   verbose appliedFilters to a single boolean. Two-arm 404 (NotFound vs
   NoFilterReport) lets dashboards distinguish "wrong URL" from "legacy review".

2. **CLI `clawreview review filter-report <reviewId>`** (`3d16f8d`) — CLI face of
   the new server endpoint. Pair-of-three with `review drift`. Text default banner
   + JSON raw body. Forwards `--slim` as `?slim=true` on the wire so CI gates that
   only need the `applied` bit get the smallest payload. New parseFilterReportConfig
   pure helper + FilterReportConfigResult discriminated union.

3. **CLI `stats --jsonl` line-delimited stream** (`0bc8e94`) — Header + 5 severity
   bucket lines (canonical SEVERITY_ORDER) + footer. Log aggregator pipelines
   (Datadog, Splunk, CloudWatch Insights) can ingest one JSON event per line. Full
   opt-in chain (--filter-summary + --json-header + --jsonl) required for back-compat.
   Severity stream always emits all 5 buckets (zero-count included) for fixed-shape
   histogram contract.

4. **CLI `review drift --base/--target --on-regression <cmd>` hook** (`88440b5`) —
   Fires when target has MORE findings than base in at least one bucket. Strictly
   narrower than `hasDrift`: bug-fix-only deltas don't fire. New
   computeRegressionSlice(drift) pure helper extracts the per-bucket positive-only
   slice. Use case: CI auto-files JIRA when a refactor regresses.

5. **Telemetry `clawreview_review_filter_report_reads_total{shape}` counter +
   route wiring** (`986640e`) — Read-shape attribution for feature 1. Closed
   full|slim set. 404 arms intentionally inert (don't inflate read rate). New
   REVIEW_FILTER_REPORT_SHAPES + deriveReviewFilterReportShape +
   observeReviewFilterReportRead helpers.
