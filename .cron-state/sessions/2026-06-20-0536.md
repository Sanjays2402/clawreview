## How they hang together

This batch closes the loop on "what does ClawReview cost and how is it
behaving" — five slices that together make the system observable
end-to-end:

- #5 estimates the cost BEFORE a review runs (and either bails or
  prints) — `preflightBudget()` mirrors the pipeline's discovery path
  (parseUnifiedDiff → filterIgnored → selectReviewableFiles → chunkFile)
  and hits a built-in per-model pricing table with conservative
  fallbacks. Wired into the worker so over-budget reviews never spend
  tokens, and into the CLI behind `--fail-on-budget`.

- #3 records what each agent ACTUALLY spent (latency, error rate,
  findings produced) — three new Prometheus series exposed via the
  existing /metrics surface, populated from `summary.agentExecutions`
  the worker already has. Histogram buckets tuned for OpenAI-compatible
  LLM calls including retry-against-rate-limit tails.

- #4 lets operators see what's queued NOW — `GET /api/internal/queue`
  returns pending/inflight totals, per-job-name breakdown, and a
  bounded ring of recent failures (default 25, newest-first). InMemory
  adapter tracks failures itself; Bull adapter pulls from BullMQ's
  `getFailed()`. Falls back to `health()` + empty arrays when the
  adapter only implements the lighter interface.

- #2 calibrates findings by confidence — floors low-confidence nits
  (< 0.35) so they stop crowding genuine low-severity issues out of the
  per-file cap, and promotes high-confidence security findings
  (≥ 0.85 → medium+, ≥ 0.95 → high+) so reviewers see real risk even
  when the model self-rated cautiously. All thresholds configurable;
  pass-through is the no-op default when `disabled: true`. Tagged with
  `calibrated:nit-floor` / `calibrated:security-bump` /
  `calibrated:security-high` so the dashboard can filter them.

- #1 lets reviewers triage PR size BEFORE running a review —
  `clawreview diff-stats` parses the diff and prints totals,
  per-language breakdown, and the top-10 largest files. Has both text
  (default) and JSON modes; accepts diff from `git diff`, `--input`, or
  stdin (`--diff -`).

## Decisions

- **Calibration runs BEFORE aggregate**, not after. That way the
  bumped security findings still compete on `max_findings_per_file`
  and the floored nits don't crowd genuine low-severity issues out of
  the cap.
- **Cost estimator runs only on the diff, no I/O.** Worth ~10ms
  even for very large PRs; the alternative (token-count via a real
  tokenizer) would force pulling tiktoken into the agents package.
- **Queue `details()` is optional on the adapter interface** so a
  caller adding a third backend later doesn't have to implement it on
  day one. The route gracefully falls back to `health()` + empty
  arrays.
- **Per-agent metrics live on the shared MetricsBundle**, not a
  parallel registry, so a single /metrics scrape catches everything.
  `observeAgentExecutions()` is a structural API (no
  `@clawreview/types` dep in telemetry).
