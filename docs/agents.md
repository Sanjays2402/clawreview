# Agents

Each agent is a small object that knows its name, its default model, and how
to turn a diff hunk into a list of findings. The shape is:

```ts
interface Agent {
  name: AgentName;
  description: string;
  defaultModel: string;
  run(input: AgentRunInput): Promise<AgentRunResult>;
}
```

## Built-in agents

| Name | Focus | Notes |
| --- | --- | --- |
| `security` | Auth, authz, injection, deserialization, crypto, race conditions, unsafe defaults | Always cites a CWE when applicable. |
| `performance` | N+1 queries, blocking IO, quadratic loops, missing indexes, large allocations | Skips micro-optimizations. |
| `style` | Naming, readability, dead code, unsafe casts, mismatched comments | Defaults to `low` or `nit` severity. |
| `accessibility` | WCAG-tier issues in UI diffs | Skips non-UI files. |
| `sql-injection` | Dynamic SQL construction, ORM raw escapes, unsafe dynamic identifiers | Only reports CWE-89 class issues. |
| `secrets` | Committed tokens, keys, PEM blocks, JWTs | Regex pre-scan, then LLM confirmation. |

## Adding a new agent

1. Implement `Agent` in `packages/agents/src/`.
2. Register it in `AGENT_REGISTRY`.
3. Add a default in `packages/types/src/config.ts` if it should ship enabled.
4. Add a vitest for the JSON schema validation of its output.

The aggregator deduplicates findings across agents, so two agents reporting
the same problem will not show up twice in the final comment.

## Prompts

Each agent owns its own system prompt. Prompts are intentionally narrow:
they describe what the agent is allowed to flag and what it should never
flag, plus a strict JSON output spec. The user template is generated by
`PromptedAgent` and includes the diff hunk, language, file path, and
optional surrounding context.
